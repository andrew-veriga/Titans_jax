# Двухфазное обучение Gemma-Titans

## Контекст

Проект исследует, можно ли заменить Global Attention слои Gemma3 1B на NeuralMemory (Titans) — блок памяти, который обучается на лету из входного потока токенов. Titans-блоки встроены в слои 5, 11, 17, 23. Цель: NeuralMemory должен накапливать долгосрочный контекст вместо дорогого Global Attention с большим окном.

---

## Фаза 1: Послойная дистилляция

### Мотивация

NeuralMemory инициализируется случайно. Прямое обучение на LM loss с нуля нестабильно: случайная память вносит шум во все последующие слои, градиенты плохо распространяются через 26 слоёв, и модель расходится.

Послойная дистилляция решает это изолированно — каждый Titans-блок обучается независимо, сравнивая свой выход с эталонным выходом соответствующего Global Attention слоя.

### Архитектура обучения

```
Вход x (от предыдущего учителя)
    │
    ├──▶ Teacher Pass (полная маска, is_teacher_mode=True)  → out_teacher
    │        └── только Attention, без Memory
    │
    └──▶ Student Pass (усечённое окно 128, is_teacher_mode=False)
             └── Attention + NeuralMemory  → out_student

Loss = log1p(MSE(out_student, stop_gradient(out_teacher)))

Следующий слой получает: x = out_teacher  ← ключевая деталь
```

### Ключевые свойства

- **Два прохода на каждый Titans-блок**: teacher (эталон) + student (обучаемый)
- **stop_gradient на входе студента**: студент не влияет на вход учителя
- **Цепочка учителей**: `x = out_teacher` — следующий слой получает чистые эмбеддинги учителя, не замутнённые студентом
- **Изолированные градиенты**: каждый Titans-слой обучается независимо, ошибки не накапливаются

### Результаты и ограничения

**Преимущества:**
- Быстро снижает лоссы (первые 15 000 шагов с batch_size=8)
- Стабильное обучение, нет расходимости
- Каждый слой получает чёткий обучающий сигнал

**Ключевое ограничение — exposure bias:**
Во время обучения студент всегда получает эмбеддинги учителя (`x = out_teacher`). Но во время инференса `out_teacher` недоступен — студент получает собственные предыдущие выходы. Это несоответствие ограничивает качество генерации.

**Наблюдаемый эффект:**
- Слой 5: `memory_gate` стабильно растёт
- Слои 11, 17, 23: `memory_gate` уходит ниже нуля, sigmoid→0, вклад памяти≈0
- Лоссы перестают снижаться, генерация остаётся невнятной несмотря на низкие distillation losses

## Фаза 2: LM Fine-tuning

### Мотивация

Послойная дистилляция обучает NeuralMemory имитировать отдельные слои, но не обучает модель как единое целое генерировать текст. После Фазы 1 веса Memory уже инициализированы осмысленно — можно переключиться на сквозное обучение с реальным языковым сигналом.

Фаза 2 устраняет оба ключевых ограничения Фазы 1: exposure bias и изолированность слоёв.

### Архитектура обучения

```
Вход x
    │
    ▼
[Слой 0-4: стандартный Gemma Block]
    │
    ▼
[Слой 5: TitansBlock, Student Pass]  → out_student_5
    │ x = out_student_5  ← студент передаёт свой выход
    ▼
[Слой 6-10: стандартный Gemma Block]
    │
    ▼
[Слой 11: TitansBlock, Student Pass]  → out_student_11
    │ x = out_student_11
    ▼
    ... (аналогично для слоёв 17, 23)
    │
    ▼
[Final Norm → Logits]
    │
Loss = CrossEntropy(logits[:, :-1, :], tokens[:, 1:])
```

### Ключевые свойства

- **Один проход**: нет teacher pass, нет дублирования вычислений
- **Сквозные градиенты**: градиент LM loss распространяется через все 26 слоёв одновременно
- **Нет stop_gradient**: студент обучается на собственных выходах (как при инференсе)
- **Реальный сигнал**: loss — предсказание следующего токена, не имитация эмбеддингов
- **Сниженные LR**: веса уже предобучены в Фазе 1 (lr_muon: 1e-4, lr_adam: 3e-4 вместо 3e-4 / 1e-3)

### Что меняется в `gemma_titans.py`

```python
# Конфиг
training_phase: int = 1  # → установить 2

# _apply_attention: для TitansBlock в training_phase==2
layer_cache_student, out_student = block(
    x,               # без stop_gradient
    ...
    is_teacher_mode=False,
    kv_seq=x_prev    # без stop_gradient
)
x = out_student      # студент ведёт цепочку

# __call__: LM loss вместо заглушки
lm_ce = optax.softmax_cross_entropy_with_integer_labels(
    logits[:, :-1, :].astype(jnp.float32),
    tokens[:, 1:]
)
layer_losses['lm_loss'] = jnp.mean(lm_ce)
```

## Сравнительная таблица

| Характеристика | Фаза 1 | Фаза 2 |
|---|---|---|
| Loss | MSE(student, teacher) per layer | CrossEntropy(logits, next_token) |
| Проходов на шаг | 2× (teacher + student) | 1× |
| Цепочка слоёв | Teacher outputs | Student outputs |
| stop_gradient | На входе студента | Нет |
| Градиентный поток | Изолирован по слоям | Сквозной, все слои |
| Exposure bias | Есть | Нет |
| LR | Полный (3e-4 / 1e-3) | Сниженный (1e-4 / 3e-4) |
| Начальные веса | Случайная инициализация Memory | Из сохранённого `saved_titans_delta` |
| Вычислительная стоимость | Высокая (2× forward) | Нормальная (1× forward + logits) |

```
    └── Каждый Titans-блок учится имитировать свой Attention-эталон
    └── Memory инициализируется с нуля → осмысленные веса
    └── Быстрое и стабильное обучение
    └── Сохранение: saved_titans_delta.zip
         │
         ▼
Фаза 2 (10 000+ шагов)
    └── Загрузка весов из Фазы 1
    └── Сквозное обучение на LM loss
    └── Модель учится генерировать связный текст
    └── Memory gate получает реальный сигнал от языкового качества
    └── Сохранение: saved_titans_phase2.zip
```
